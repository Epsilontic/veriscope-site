What it is

Veriscope detects narrowing in internal representation structure that precedes brittle behavior‚Äîwhile external metrics still look fine. Two independent channels (streaming geometry + fragility) feed a preregistered sequential test; all decisions are logged for deterministic, third-party replay.

Install (dev)

python -m venv .venv
source .venv/bin/activate
pip install -e .
# Install PyTorch for your platform:
# CPU  : pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
# CUDA : pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

Optional extras: ripser for H0 persistence; pyarrow/fastparquet for Parquet.

Quick start

CLI help / version (fast, no heavy imports):

veriscope --help
veriscope --version

Smoke run (fast end-to-end):

export SCAR_SMOKE=1
export SCAR_OUTDIR=./out_scar_smoke
export SCAR_DATA=./data
veriscope

Full sweep (more seeds/epochs):

unset SCAR_SMOKE
export SCAR_OUTDIR=./out_scar_full
export SCAR_DATA=./data
veriscope

Config knobs

  • SCAR_OUTDIR ‚Äî artifact root
  • SCAR_DATA ‚Äî dataset root (CIFAR-10 / STL-10)
  • SCAR_SMOKE=1 ‚Äî tiny budgets; at least one heavy pass
  • SCAR_EVAL_SPLIT ‚Äî eval | calib | both
  • SCAR_NUM_WORKERS ‚Äî DataLoader workers (recommend 0 for determinism)
  • Gate: SCAR_FAMILY_Z_THR, SCAR_WARN_CONSEC, SCAR_FAMILY_WINDOW
  • Soft-GT: SCAR_FIXED_GT_RANK_MIN, SCAR_SOFT_Q

Determinism: sets CUBLAS_WORKSPACE_CONFIG if unset; JL projections use a bounded, seeded cache.

Outputs & artifacts

  • Run logs: bundle_runs_{tag}.parquet (+ CSV fallback), errors log
  • Detector / calibration: learned_detector_*.json, tau_mapping_*.json, PH/JL adequacy
  • Baselines & GT: robust gradient cutoffs, baseline event CSVs
  • Overlays & summaries: overlay Parquet, summary CSVs
  • Provenance: env.json, repro.json, pip_freeze.txt, window provenance

Writes are atomic where possible; Parquet ‚ûú CSV fallback is automatic.
